Runscope has been helping thousands of developers and platform providers with robust cloud-based API testing, monitoring and debugging tools. Users of Runscope Radar have been able to created automated API tests for Internet-accessible backend services, complete with assertions and multiple testing locations around the world. However, many companies have APIs that live solely behind a firewall, only accessible from within the private network.

Runscope now offers the Runscope Radar On-premises Agent, bridging Runscope to private APIs not available over the public internet. By running the agent within a private infrastructure, Runscope Radar can now monitor and test against endpoints that are otherwise inaccessible through the internet. The agent is particularly useful for testing or monitoring APIs in the following situations:

  • Monitoring private or internal APIs that reside behind a firewall
  • Testing APIs on localhost or in a private staging environment
  • Accessing APIs that require a fixed client IP address (IP whitelisting)

Runscope Radar uses a hybrid on-premises approach. The agent runs on a host within the private infrastructure, but test data is managed and stored in the Runscope cloud. The Runscope dashboard serves as a single interface for defining tests, viewing results, configuring notifications and enabling 3rd-party integrations. All communication between the agent and Runscope cloud is made securely via HTTPS. The agent requests work to be processed and sends results back over a TLS-encrypted connection and is not externally addressable. The agent is available for download, and can be run natively on the following operating systems: Mac OS X, Windows (32-bit / 64-bit) and Linux (32-bit / 64-bit).
